How LoRAs are matched

Every 24 hours we sync the LoRAs from CivitAI that are tagged character and built on the Anima base model, then try to link each one to a character here.

Matching is word-set based: both the LoRA's title, tags and trigger words and our character names are lower-cased, accent-stripped and split into tokens. A LoRA is linked to a character when the LoRA's tokens cover (or exactly equal) that character's name tokens.

Tips for LoRA creators

  • Tag the LoRA with character on CivitAI.
  • Use the Anima base model so the sync picks it up.
  • Put the danbooru-style name in the title or as a trigger word — e.g. hatsune miku, vocaloid.
  • For ambiguous names add the series in parentheses, e.g. Yuki (To-Love-Ru).
  • Short, clear titles match better than ones buried under long stylistic prefixes.

Heads up: matching is fuzzy — a LoRA can sometimes end up linked to the wrong character, especially for short common names ("Yuki", "Akari"). If you spot a mismatch, please report it via the contact form in the footer.
